<purpose>
Index the codebase with CodeGraphWiki (code-graph-builder MCP) to build a persistent knowledge graph,
vector embeddings, and wiki pages. The knowledge graph and embeddings are stored in the CGB workspace
and queried at runtime by gsd-phase-researcher and gsd-executor via MCP tools. The wiki pages are
written to .planning/codebase/wiki/ as static reference docs for planning.
</purpose>

<philosophy>
**Why CodeGraphWiki instead of parallel mapper agents:**
- Builds a queryable knowledge graph (call graphs, dependencies) — not just static text
- Vector embeddings enable semantic search at runtime ("find auth middleware")
- Single indexing pass covers everything; no token cost in the planning context window
- Wiki pages are generated by LLM with full codebase context — higher quality than grep-based analysis
- Knowledge graph persists across sessions; incremental re-index is fast
</philosophy>

<process>

<step name="init_context" priority="first">
Load codebase mapping context:

```bash
INIT=$(node ~/.claude/get-shit-done/bin/gsd-tools.cjs init map-codebase)
```

Extract from init JSON: `commit_docs`, `codebase_dir`, `codebase_dir_exists`.

Also capture the current working directory as the repo path:
```bash
REPO_PATH=$(pwd)
```
</step>

<step name="check_existing">
Check if .planning/codebase/wiki/ already exists.

```bash
ls -la .planning/codebase/wiki/ 2>/dev/null
```

**If wiki/ exists and has pages:**

```
.planning/codebase/wiki/ already exists with these pages:
[list files]

What's next?
1. Rebuild - Re-index from scratch (force-rebuilds graph, embeddings, and wiki)
2. Refresh wiki - Regenerate wiki pages only (graph and embeddings reused)
3. Skip - Use existing index as-is
```

Wait for user response.

If "Rebuild": continue to run_indexing with `rebuild=true`
If "Refresh wiki": continue to pull_wiki_pages (skip run_indexing, call generate_wiki instead)
If "Skip": exit workflow

**If wiki/ doesn't exist:**
Continue to run_indexing.
</step>

<step name="run_indexing">
Index the repository. This builds the knowledge graph, vector embeddings, and wiki pages.

**This takes 2-10 minutes depending on repo size.** Inform the user before calling:

```
Indexing codebase at {REPO_PATH}...
This builds the knowledge graph, vector embeddings, and wiki pages.
Estimated time: 2-10 minutes depending on repo size.
```

Call the MCP tool:

```
mcp__code-graph-builder__initialize_repository(
  repo_path="{REPO_PATH}",
  rebuild=<true if user selected Rebuild, false otherwise>,
  wiki_mode="comprehensive"
)
```

Wait for completion. The tool returns a summary with graph stats and wiki page count.

If the tool returns an error:
- "Repository not indexed" or similar → check that code-graph-builder MCP is configured in ~/.claude/settings.json
- LLM/embedding errors → check API keys in MCP server env config
- Report error to user with actionable next steps

Continue to pull_wiki_pages on success.
</step>

<step name="pull_wiki_pages">
Pull wiki pages from CodeGraphWiki and write them to .planning/codebase/wiki/.

**Create directory:**
```bash
mkdir -p .planning/codebase/wiki
```

**List available pages:**
```
mcp__code-graph-builder__list_wiki_pages()
```

**For each page in the list, fetch and write:**
```
page_content = mcp__code-graph-builder__get_wiki_page(page_id="{page_id}")
```

Write content to `.planning/codebase/wiki/{page_id}.md` using the Write tool.

Also fetch and write the index page:
```
index_content = mcp__code-graph-builder__get_wiki_page(page_id="index")
```
Write to `.planning/codebase/wiki/index.md`.

Continue to write_index.
</step>

<step name="write_index">
Write .planning/codebase/INDEX.md — a quick-reference guide for GSD planning agents.

**Get repo info for stats:**
```
mcp__code-graph-builder__get_repository_info()
```

**Write the file:**

```markdown
# Codebase Index

**Indexed:** {date}
**Repository:** {REPO_PATH}
**Graph:** {node_count} nodes, {relationship_count} relationships
**Wiki pages:** {page_count}

## Wiki Pages

{for each page: - [{page_title}](wiki/{page_id}.md)}

## How to Query the Codebase

The knowledge graph and vector embeddings are available via the code-graph-builder MCP tools.

**Find existing implementations (semantic search):**
Use `mcp__code-graph-builder__semantic_search` with a natural language query.

**Find APIs with full call graph:**
Use `mcp__code-graph-builder__find_api` — returns signatures + callers/callees.

**Understand relationships:**
Use `mcp__code-graph-builder__query_code_graph` for natural language graph queries.

**Get exact source location:**
Use `mcp__code-graph-builder__locate_function` to get file path + line numbers.

## Refreshing the Index

Run `/gsd:map-codebase` after significant code changes to keep the index current.
To rebuild from scratch: `/gsd:map-codebase rebuild`
```

Continue to scan_for_secrets.
</step>

<step name="scan_for_secrets">
**CRITICAL SECURITY CHECK:** Scan output files for accidentally leaked secrets before committing.

Run secret pattern detection on all written files:

```bash
grep -rE '(sk-[a-zA-Z0-9]{20,}|sk_live_[a-zA-Z0-9]+|sk_test_[a-zA-Z0-9]+|ghp_[a-zA-Z0-9]{36}|gho_[a-zA-Z0-9]{36}|glpat-[a-zA-Z0-9_-]+|AKIA[A-Z0-9]{16}|xox[baprs]-[a-zA-Z0-9-]+|-----BEGIN.*PRIVATE KEY|eyJ[a-zA-Z0-9_-]+\.eyJ[a-zA-Z0-9_-]+\.)' .planning/codebase/ 2>/dev/null && SECRETS_FOUND=true || SECRETS_FOUND=false
```

**If SECRETS_FOUND=true:**

```
⚠️  SECURITY ALERT: Potential secrets detected in codebase documents!

Found patterns that look like API keys or tokens in:
[show grep output]

This would expose credentials if committed.

**Action required:**
1. Review the flagged content above
2. If these are real secrets, they must be removed before committing
3. Consider adding sensitive files to Claude Code "Deny" permissions

Pausing before commit. Reply "safe to proceed" if the flagged content is not actually sensitive, or edit the files first.
```

Wait for user confirmation before continuing to commit_codebase_map.

**If SECRETS_FOUND=false:**
Continue to commit_codebase_map.
</step>

<step name="commit_codebase_map">
Commit the codebase index:

```bash
node ~/.claude/get-shit-done/bin/gsd-tools.cjs commit "docs: index codebase with CodeGraphWiki" --files .planning/codebase/
```

Continue to offer_next.
</step>

Present completion summary and next steps.

**Get wiki page count:**
```bash
ls .planning/codebase/wiki/*.md 2>/dev/null | wc -l
```

**Output format:**

```
Codebase indexed successfully.

Knowledge graph + embeddings: ready for semantic queries
Wiki pages written to .planning/codebase/wiki/:
[list each page title from INDEX.md]

---

## ▶ Next Up

`/gsd:new-project`

<sub>`/clear` first → fresh context window</sub>

---

**Also available:**
- Re-index after changes: `/gsd:map-codebase`
- Force full rebuild: `/gsd:map-codebase rebuild`
- Browse wiki: `cat .planning/codebase/INDEX.md`

---
```

End workflow.
</step>

</process>
